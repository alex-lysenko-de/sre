# Структура каталогов `src/`

```
src/
├── components/       — переиспользуемые Vue-компоненты (модалки и т.п.)
├── composables/      — логика/данные (Supabase-запросы), см. [[Обзор-composables]]
├── modules/          — конфигурация, хелперы (например src/modules/storage.js)
├── views/            — страницы приложения, см. [[Карта-маршрутов]]
├── router/            — конфигурация Vue Router (src/router/index.js)
├── stores/           — Pinia-модули (user.js, config.js)
supabase/
└── functions/        — Edge Functions (Deno/TypeScript), см. [[Обзор-Edge-Functions]]
```

## `src/components/`

Переиспользуемые компоненты, используемые несколькими views:
`AddEditChildModal.vue`, `BusChangeModal.vue`, `BusDetailModal.vue`,
`ChildPresenceModal.vue`, `GroupChangeModal.vue`, `ResetHistoryPanel.vue`.
`BusChangeModal.vue`/`GroupChangeModal.vue` — индикаторы-переключатели
группы/автобуса, встроенные в `App.vue` (см. [[Первая-регистрация-дня]]).

## `src/composables/`

Функциональные модули, обёртывающие обращения к Supabase. Каждый экспортирует
функцию `useXXX()`, возвращающую объект методов (см. паттерн в
[[Обзор-composables]]): `useArmband.js`, `useBusData.js`, `useChildPresence.js`,
`useChildren.js`, `useDays.js`, `useGroups.js`, `useScan.js`, `useSupabaseUser.js`,
`useUser.js`.

## `src/modules/`

`storage.js` — единственный модуль на текущий момент, инкапсулирует
LocalForage-хранилище для auth-данных (см. [[Кэширование-LocalForage]]).

## `src/views/`

Страницы приложения — по одной на маршрут, см. полный список в
[[Карта-маршрутов]].

## `src/router/`

`index.js` — единственная точка регистрации маршрутов и глобальный
navigation guard (`router.beforeEach`), см. [[Карта-маршрутов]].

## `src/stores/`

Pinia store'ы: `user.js` (см. [[stores-user]]), `config.js` (общая
конфигурация — `total_groups`, `total_buses`, `year`, `base_url` и т.д., см.
[[config]]).

## `supabase/functions/`

Edge Functions на Deno/TypeScript: `auth`, `delete-user`, `invite-accept`,
`invite-generate`. См. [[Обзор-Edge-Functions]].

## Замечание

В репозитории нет каталога `supabase/migrations/` — источником истины по
схеме БД служат файлы в `doc/` (`table_structure.md`, `db_triggers.sql`,
`doc/db/*.sql`), применяемые вручную через Supabase SQL Editor. См.
[[Обзор-схемы-БД]].
