# Учёт автобусов

> Источник: `doc/info.md`, `doc/AdminBusView_TechTask.md`, `doc/db/adminBusView.md`.

Автобусы — отдельная часть системы. Во время посадки может использоваться
режим быстрого сканирования: руководитель группы сканирует всех детей в
своём автобусе; система автоматически определяет фактический состав
каждого автобуса. Если ребёнок пересел в другой автобус, актуальным
считается его последнее по времени сканирование.

Автобусов как отдельной таблицы в БД нет — только порядковые номера
`1..config.total_buses` (см. [[Группы-и-рабочий-день]], [[config]]).

## Подсчёт детей в автобусе

Источник — [[children_today]]. Ребёнок учитывается в автобусе, если:

```sql
WHERE bus_now IS NOT NULL AND presence_now > 0
```

## Подсчёт Betreuer в автобусе

Источник — [[user_group_day]] JOIN [[users]]. Betreuer учитывается, если:

```sql
WHERE day = :текущая_дата AND "isPresentToday" = 1 AND bus_id IS NOT NULL
```

## Реализация (AdminBusView)

`AdminBusView.vue` (маршрут `/admin-busses`, admin-only) агрегирует эти два
запроса через composable [[useBusData]] (`fetchBusData()`,
`fetchBusChildren()`, `fetchBusBetreuer()`) и предоставляет кнопки
управления днём — «Tag starten» / «Soft Reset» / «Tag schließen»,
создающие записи в [[reset_events]] (см. [[Триггеры]] для того, что при
этом происходит с [[children_today]]/[[groups_today]]).
