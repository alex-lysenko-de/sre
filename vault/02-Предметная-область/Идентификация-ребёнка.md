# Идентификация ребёнка

> Источник: `doc/info.md`, `doc/QR-codes.md`, `doc/ArmbandTask.md`.

Каждый ребёнок получает браслет с QR-кодом. Браслет **не является именным** —
содержит только уникальный идентификатор (`band_id`). Браслет можно заменить:
в БД изменяется только привязка нового `band_id` к ребёнку в таблице
[[children]], сам ребёнок остаётся тем же самым. Это позволяет быстро
заменять повреждённые/потерянные браслеты без изменения остальных данных.

## QR-код как способ идентификации

После сканирования система мгновенно определяет ребёнка, его группу и
текущий статус участия. Если камера недоступна, руководитель отмечает
ребёнка вручную (см. [[Учёт-автобусов]], [[Переклички]]).

## Текущий формат публичного URL

```
https://sre-nrw.de?id={id}
```

Роутер (`src/router/index.js`) перехватывает `?id=` **и** `?n=` в
query-параметрах любого маршрута и перенаправляет на `/armband/:id` (см.
[[Карта-маршрутов]], [[QR-код-URL]]).

## Сценарий привязки/использования браслета

1. `/armband/:id` (`ArmbandView.vue`) — запрос `children` по `band_id = id`
   через [[useArmband]].`getBraceletStatus()`.
2. Если ребёнок **не найден** — форма привязки браслета
   (`ArmbandConnectView.vue`): выбор ребёнка из своей группы
   (`useArmband.getChildrenByGroup()`), кнопка «Armband zuordnen» →
   `assignBraceletToChild()`.
3. Если **найден** — переход на `/child/:id` (`ChildDetailView.vue`):
   имя/возраст/группа, действия «Bearbeiten», «Anwesend markieren»,
   «Zurück zur Gruppe».
