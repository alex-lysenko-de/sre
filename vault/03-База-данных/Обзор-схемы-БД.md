## Находка реверс-инжиниринга

Формулировка тикета 108.md упоминает таблицы `groups`, `buses` — **их не
существует**. Группы и автобусы — не отдельные сущности, а просто
smallint-поля (`children.group_id`, `scans.bus_id` и т.п.). Единственная
агрегирующая по группам таблица — [[groups_today]]; число групп/автобусов
хранится как значение в [[config]] (`total_groups`, `total_buses`). См.
[[Группы-и-рабочий-день]], [[Учёт-автобусов]].

## Таблицы

| Таблица | Назначение |
|---|---|
| [[children]] | Справочник детей: имя, возраст, группа, привязанный браслет |
| [[scans]] | История всех сканирований браслетов (append-only) |
| [[children_today]] | Оперативное состояние присутствия ребёнка сегодня (upsert-таблица, источник истины для [[Переклички]]) |
| [[groups_today]] | Агрегированные счётчики присутствия по группам на сегодня |
| [[reset_events]] | Журнал операций сброса/открытия дня (0/1/2), запускает массовый пересчёт |
| [[users]] | Пользователи приложения (admin/user), связаны с `auth.users` |
| [[user_group_day]] | Ежедневная привязка Betreuer к группе/автобусу + статус присутствия |
| [[config]] | Общая конфигурация приложения (key/value) |
| [[days]] | Список дней мероприятия (расписание) |
