# `reset_events`

> Источник: `doc/table_structure.md`, `doc/db_triggers.sql`.

Журнал операций массового сброса/открытия дня. Каждая вставка запускает
триггер `on_reset_event_insert`, который сразу пересчитывает
[[children_today]] и [[groups_today]] — сам по себе `reset_events` не
хранит счётчиков, только историю событий (используется также для
отображения в `ResetHistoryPanel.vue`).

```sql
create table public.reset_events (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  user_id smallint null,
  day character varying null,  -- date in ISO format yyyy-mm-dd
  event_type smallint null default '1'::smallint,
  constraint reset_events_pkey primary key (id)
);
```

## `event_type`

| Значение | Название | Что происходит |
|---|---|---|
| `0` | Total reset («Tag schließen») | `DELETE FROM children_today; DELETE FROM groups_today;` — полная очистка |
| `1` | Normal reset («Tag starten») | Первый reset дня: `groups_today.children_today = children_now`, затем `children_now = 0`; последующие resets того же дня — только `children_now = 0`. В обоих случаях `children_today.presence_now = 0` |
| `2` | Soft reset | Только `presence_now`/`children_now` обнуляются, без сохранения снимка |

## Кто пишет

- [[useDays]] — `startNewDay()` (type=1), `softReset()` (type=2),
  `closeDay()` (type=0)
- Кнопки «Tag starten» / «Soft Reset» в `AdminBusView.vue` (через
  [[useBusData]]/[[useDays]])
