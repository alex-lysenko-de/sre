# Обзор Edge Functions

> Источник: перечень каталогов `supabase/functions/` (`auth`, `delete-user`,
> `invite-accept`, `invite-generate`); `doc/genkeys_curl.bat` (пример вызова
> `invite-generate`, оставлен как есть в тикете 107 — соответствует коду,
> секретов не содержит). Исходники самих функций (Deno/TypeScript) в рамках
> этой ревизии (тикет 108) не открывались построчно — описание ниже даётся
> на уровне назначения по имени каталога и связанным views/документации, а
> не построчного разбора кода.

Edge Functions выполняются на стороне Supabase (Deno) — используются для
операций, которые не должны или не могут выполняться напрямую с клиента
через обычный Supabase JS SDK (административные действия с `auth.users`,
секреты вроде `service_role`-ключа).

## Функции

| Функция | Предполагаемое назначение | Связанный UI |
|---|---|---|
| `auth` | Серверные операции аутентификации, выходящие за рамки стандартного `supabase.auth.*` на клиенте | — |
| `delete-user` | Удаление пользователя (требует `service_role`, недоступного клиенту) | `UsersView.vue` (`/users-edit`, admin-only) |
| `invite-generate` | Генерация приглашения для нового пользователя | `InviteGeneratorView.vue` (`/invite`, admin-only) |
| `invite-accept` | Принятие приглашения, создание учётной записи | `InviteAcceptView.vue` |

`doc/genkeys_curl.bat` — рабочий пример прямого вызова `invite-generate`
через `curl`, подтверждённый в тикете 107 как соответствующий коду функции.

## Секреты

Развёртывание и секреты Edge Functions (в частности реальный
`service_role`-ключ) описаны в `doc/sb_install_keys.bat` /
`doc/supabase_install.md` — оба файла содержат реальный ключ в открытом
виде (зафиксировано как проблема безопасности в
`tickets/107/AUDIT_REPORT.md`, не устранено по прямому указанию владельца
проекта на момент той ревизии). Эта заметка **не цитирует** значение ключа
— только упоминает факт и его расположение, см. `_Конвенции.md`, пункт 6.
